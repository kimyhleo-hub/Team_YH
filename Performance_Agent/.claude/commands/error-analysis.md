오류 분석을 수행한다. 새 실험을 제안하기 전에 반드시 이 과정을 완료한다.

1. `context.md`를 읽어 현재 성능 수치와 최근 실험 이력을 파악한다.

2. 프로젝트 폴더에서 검증 예측 결과 파일(OOF 예측, validation 결과 등)을 찾는다.
   찾지 못하면 사용자에게 파일 경로를 요청한다.

3. 다음 분석을 수행한다:
   - 클래스별 오류율 (분류 문제)
   - 오류 샘플의 신뢰도 분포 (고신뢰 오류 vs. 경계선 오류)
   - 특정 피처 구간 또는 조건에 오류가 집중되는지 확인

4. 발견한 패턴을 아래 중 하나로 진단한다:
   - 데이터 부족 → 해당 케이스 데이터 수집·증강
   - 레이블 오류 → 데이터 정제
   - 피처 부재 → 피처 엔지니어링
   - 모델 표현력 부족 → 아키텍처 개선
   - 분포 불일치 → 검증 전략 재설계

5. 진단 결과를 `context.md`에 기록하고, 진단에 근거한 다음 실험 방향을 제시한다.

수치 비교만으로 방향을 결정하지 않는다. 오류 패턴 근거 없는 제안은 하지 않는다.
