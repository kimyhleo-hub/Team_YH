새 공모전 또는 연구 프로젝트를 설정한다.

## 1단계: 정보 수집

사용자에게 아래를 순서대로 확인한다.

1. 프로젝트 이름 (폴더명으로 사용, 영문·하이픈 권장)
2. 태스크 유형 (이진 분류 / 다중 분류 / 회귀 / 이미지 분류 / NLP / 기타)
3. 평가지표 (F1, AUC, RMSE, Accuracy, mAP 등)
4. 제출 형식 및 제약 조건 (추론 시간, 제출 횟수, 모델 크기 등)
5. 베이스라인 모델과 초기 성능 (없으면 "미정"으로 기록)
6. 데이터셋 개요 (크기, 클래스 수, 특이사항)

## 2단계: 폴더 및 파일 생성

수집한 정보로 `projects/{프로젝트이름}/` 폴더를 생성하고 두 파일을 작성한다.

### CLAUDE.md 작성

```markdown
# {프로젝트명} — Performance Agent 확장

## 프로젝트 개요

- **태스크 유형**: {태스크}
- **평가지표**: {평가지표}
- **제출 형식**: {형식}
- **제약 조건**: {제약}

## 현재 베이스라인

- **모델**: {모델}
- **성능**: {성능}
- **주요 미해결 문제**: (분석 전 — 미정)

## 실행 명령어

학습 : python train.py
평가 : python evaluate.py --fold all
예측 : python predict.py

## 프로젝트 특화 전략

**효과 확인된 기법**: (아직 없음)
**효과 없었던 기법**: (아직 없음)
**우선 탐색 방향**: (오류 분석 후 결정)

## 참조

- 실험 이력: `context.md`
- 검증 전략: `../../strategies/validation_strategy.md`
- 하이퍼파라미터: `../../strategies/hyperparameter.md`
- 오류 분석: `../../strategies/error_analysis.md`
```

### context.md 작성

