# Hw_Agent Plan

Last updated: 2026-05-17

기계공학 과제를 처리하는 Agent를 작은 단계로 계획하고, 검증하며, 재사용 가능한 형태로 정리하기 위한 운영 계획서.

## 1. 목표

- 문제를 읽고 요구사항을 구조화한다.
- 풀이, 코드, 그래프, 해석을 제출 가능한 형태로 정리한다.
- 실행 가능 항목은 검증하고, 실행 불가 항목은 명시적으로 분리한다.
- 과목별 규칙은 분리하되 공통 운영 흐름은 일관되게 유지한다.

## 2. 운영 원칙

- 모든 큰 변경은 먼저 계획부터 세운다.
- 한 세션에서 한 단계만 확실히 끝낸다.
- 작은 변경 후 바로 검증한다.
- 검증이 끝난 뒤 다음 단계로 넘어간다.
- 공통 규칙은 `AGENT.md`, 과목별 규칙은 각 과제 폴더의 `notes.md`에 둔다.

## 3. 기본 워크플로우

```text
계획 수립
  -> 현재 단계 범위 확정
  -> 해당 단계만 구현
  -> 테스트/검증
  -> 결과 정리
  -> TODO.md 갱신
  -> 다음 단계 계획
```

## 4. 세션 운영 방식

### 계획 단계
- 이번 세션의 목표를 한 줄로 정의한다.
- 수정할 파일, 기대 결과, 검증 방법을 먼저 정한다.
- 범위가 크면 반드시 단계로 쪼갠다.

### 실행 단계
- 현재 단계와 직접 관련된 파일만 수정한다.
- 한 번에 여러 목표를 섞지 않는다.
- 새 작업이 생기면 현재 작업과 분리해 `TODO.md`에 적는다.

### 종료 단계
- 끝낸 것, 남은 리스크, 다음 시작점을 기록한다.
- 다음 세션이 바로 이어받을 수 있게 `TODO.md`를 갱신한다.

## 5. 단계 분할 규칙

- 큰 기능을 바로 끝내려 하지 않는다.
- `계획 -> 1단계 구현 -> 검증 -> 다음 단계` 순서를 유지한다.
- 각 단계는 독립적으로 검증 가능해야 한다.
- 종료 조건이 명확하지 않으면 단계가 너무 크다고 본다.

예시:

1. 공통 운영 문서 작성
2. 과목별 `notes.md` 구조 정리
3. `solution.md` 생성 흐름 정의
4. MATLAB 과제용 실행/폴백 흐름 검증
5. 실제 과제 하나로 end-to-end 테스트

## 6. TODO.md 운영 규칙

- 프로젝트 루트에 `TODO.md`를 두고 작업 단위를 추적한다.
- 새 작업은 체크리스트 항목으로 적는다.
- 현재 세션에서 처리하지 않을 일도 바로 적는다.
- 세션 종료 전 완료/보류 상태를 반영한다.

## 7. 검증 루프

```text
작은 변경
  -> 테스트 또는 실행 검증
  -> 형식/구조 점검
  -> 문서 반영
  -> 다음 변경
```

### 검증 우선순위
- 문제 요구사항 충족 여부
- 단위와 변수 정의 일관성
- 코드 실행 가능 여부
- 그래프/출력 해석의 타당성
- 제출 문서 형식 적합성

## 8. 과제 유형별 처리 흐름

### 문제풀이형
```text
문제 원문 확보
  -> 문제 번호/조건/구해야 할 값 추출
  -> 공식 및 이론 정리
  -> 단계별 풀이 작성
  -> 단위/최종값 점검
  -> solution.md 정리
```

### Python형
```text
요구사항 정리
  -> 코드 작성
  -> 로컬 실행 또는 문법 검증
  -> 핵심 출력 확인
  -> solution.md 또는 src 정리
```

### MATLAB형
```text
문제 구조화
  -> 변수/파라미터 정의
  -> MATLAB 코드 작성
  -> 실행 가능 시 그래프 검증
  -> 실행 불가 시 폴백 기록
  -> solution.md 정리
```

## 9. 공통 문서 최소 세트

- `AGENT.md`: 공통 운영 규칙
- `Plan.md`: 전체 운영 흐름
- `TODO.md`: 현재 작업 큐
- `MATLAB_WORKFLOW.md`: MATLAB 과제 표준 절차
- `{subject}_{assignment}/notes.md`: 과제별 특수 규칙
